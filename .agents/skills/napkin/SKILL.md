## Workflow

1. Read [`../../../docs/napkin/napkin.md`](../../../docs/napkin/napkin.md) before any other repo work in the session.
2. Apply the napkin silently unless the user explicitly asks for proof that you read it.
3. Update [`../../../docs/napkin/napkin.md`](../../../docs/napkin/napkin.md) during the session when you learn a durable correction, preference, failure pattern, or winning pattern.
4. Before handoff, update [`../../../progress.md`](../../../progress.md) `## Napkin Evidence` for the most recently closed session.
5. `## Napkin Evidence` must either link [`../../../docs/napkin/napkin.md`](../../../docs/napkin/napkin.md) or say `No durable napkin update.` exactly.
6. Do not use the napkin as a progress tracker; numbered specs plus [`../../../progress.md`](../../../progress.md) own session state.
7. When another repo skill also applies, do napkin first, then continue with that skill.
